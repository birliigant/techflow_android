# Apache Answer 详细接口格式文档

这份文档补充了每个接口的请求参数、请求体模型、成功响应模型。整体以当前仓库中的实际路由注册为准，并结合现有 `docs/swagger.json` 自动整理。

## 文档合并说明

本文已合并旧版接口文档中仍需保留的对接信息：

- 统一响应壳、分页结构、认证头和访问控制分层请先看 [接口文档-入口.md](接口文档-入口.md)
- Swagger 注解与实际路由存在偏差的接口，已在对应接口位置通过“备注”说明
- Swagger 未覆盖但代码已生效的接口，已集中补充在“Swagger 未覆盖但代码已生效的接口”章节
- 如果本文与 `swagger.yaml` / `swagger.json` 出现差异，优先以当前代码实际注册路由和本文备注为准

## 公共响应壳

绝大多数 JSON 接口都返回统一结构：

```json
{
  "code": 200,
  "reason": "success",
  "msg": "Success",
  "data": {}
}
```

其中 `data` 的具体格式由每个接口自己的响应模型决定。
